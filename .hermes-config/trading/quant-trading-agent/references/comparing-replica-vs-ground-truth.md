# Comparing Indicator Replica Against Ground-Truth CSV

## Session Context (May 22, 2026)

The user exported a CSV from Ferocious Fatcat's "Single Direction" mode running on Binance BTCUSDT Perpetual. The CSV contained 21 entry signals (L1–L21) over Oct 7 – Dec 15, 2025 on 5m timeframe.

The goal: compare these FATCAT entries against our BAMBAM replica to validate parameter tuning.

## Methodology

### Step 1: Source the OHLCV data

Binance Futures API blocked from VPS (HTTP 451). Bitstamp worked:

```python
url = "https://www.bitstamp.net/api/v2/ohlc/btcusd/"
params = {"step": 300, "limit": 1000, "end": timestamp}
# Paginate backward: each page's oldest timestamp - 1 = next page's end
# Returns: [timestamp, open, high, low, close, volume]
```

27,000 bars fetched for Sep 28 – Dec 31, 2025.

### Step 2: Align CSV timestamps

FATCAT CSV timestamps (e.g. `2025-10-07 16:00`) matched exactly to 5m bar boundaries in Bitstamp data. Price alignment was within $25–$813 (Bitstamp vs Binance price variance). Good enough for signal comparison.

### Step 3: Run BAMBAM across parameter grid

Tested 18 combinations: lessRatio ∈ {0.03, 0.01, 0.005} × volMultiplier ∈ {1.5, 2.0, 2.5} × deltaMode ∈ {OR, AND}.

BAMBAM signal logic replicated in pandas:
```python
df["ph"] = df["high"].shift(1)           # offset=1 (confirmation bar)
df["pl"] = df["low"].shift(1)
df["vd"] = (df["vb"] - df["vs"]).shift(1)
df["vol"] = (df["vb"] + df["vs"]).shift(1)
df["top"] = df["ph"].rolling(200).max()
df["btm"] = df["pl"].rolling(200).min()
df["avg_vol"] = df["vol"].rolling(200).mean()
df["vd_ratio"] = abs(df["vd"] / df["vol"])
df["is_less_ratio"] = df["vd_ratio"] < lessRatio
df["high_vol"] = df["vol"] > (df["avg_vol"] * volMultiplier)
df["bull_delta"] = is_less_ratio | (vd > 0)     # OR mode
df["bull_raw"] = (pl <= btm) & high_vol & bull_delta
df["bull_signal"] = df["bull_raw"].shift(-1)     # confirmation bar offset
```

### Step 4: Compare

| Metric | Result |
|---|---|
| Exact bar match (same 5m timestamp) | **0/21 across ALL 18 combos** |
| ±1 bar match | 1/21 (only lr=0.03, vm=1.5, OR) |
| ±5 bars match | 11/21 at best (OR mode, any lessRatio/volMult) |
| AND mode total signals | 2–8 signals (far too few vs 21 FATCAT entries) |
| OR mode total signals | 129–193 signals (far too many vs 21 FATCAT entries) |

### Step 5: Diagnose

The zero-match result is not a parameter problem — it's a logic mismatch. The FATCAT "Single Direction" entries are generated by DIFFERENT signal logic than the core reversal triangles.

Evidence:
- Only 3/21 entries are at 200-bar lows
- Volume conditions vary from 0.19× to 4.81× average (many below BAMBAM's 1.5× threshold)
- Entries include bounce retracements (L6, L18, L20) — these are NOT swing lows
- The sequential L1→L21 labeling suggests a position-building grid, not independent signal detections

The core FATCAT reversal signals (triangles on the chart) are what BAMBAM targets. The "Single Direction" mode is a separate module within the FATCAT indicator package that builds grid positions using different rules (possibly ATR/percentage-based levels, not swing lows).
